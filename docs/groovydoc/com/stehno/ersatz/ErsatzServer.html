      @groovy.transform.CompileStatic
@groovy.util.logging.Slf4j
class ErsatzServer
extends java.lang.Object

      The main entry point for configuring an Ersatz server, which allows configuring of the expectations and management of the server itself. This is the class that should be instantiated in unit tests. The server will be started on an ephemeral port so as not to collide with itself or other server applications running in the test environment. In your tests, you can retrieve the server port or URL using the getPort() and getServerUrl() methods respectively. Using the ErsatzServer follows the workflow:

      1. Create the ErsatzServer instance.
      2. Configure the expectations.
      3. Start the server
      4. Run your client tests against the server.
      5. Verify the expectations.
      6. Stop the server.
      See the User Guide for more detailed information.
